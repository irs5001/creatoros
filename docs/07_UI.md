# UI / UX Principles

## 1. Content First

Creators create content.

SimpleDee helps creators automatically deliver the right thing to the right person after they engage with that content.

---

## 2. Confidence

The UI should make creators feel confident that everything is working.

Users should rarely wonder:

- Did my automation run?
- Is Instagram connected?
- Did people receive my message?

The application should proactively communicate system health.

---

## 3. Simplicity

The platform should expose only the options needed for the current task.

Advanced functionality should remain available without overwhelming first-time users.

---

## 4. Never Ask Unnecessary Questions

When possible:

Show options.

Don't interview users.

Allow exploration.

---

## 5. Progressive Disclosure

Beginner users should see only what they need.

Power users should discover more capability naturally.

---

## 6. Speak the Creator's Language

Avoid technical terminology.

Examples:

"Only If..."

instead of

"Conditions"

"Send Message"

instead of

"Delivery"


## 7. Reuse Before Recreate

Creators should rarely need to create the same thing twice.

Messages, resources, products, recommendations, and creator hubs should all be reusable throughout the application.

Editing a reusable object should update every campaign or location that references it.


## 8. Customize the Brand, Not the Experience

Creators should be able to personalize their public-facing pages through branding such as logos, colors, cover images, and basic styling.

The overall layout and navigation should remain consistent so visitors immediately know how to use every Creator Hub.


## 9. Explain Relationships

Reusable objects should always make it clear where they are being used.

Users should never wonder:

- Which campaigns use this message?
- What creator hubs include this resource?
- Which products reference this recommendation?

Whenever possible, the application should answer these questions directly.


## 10. Campaigns Deliver. Creator Hubs Organize.

Campaigns are responsible for delivering content to people.

Creator Hubs are destinations where creators organize their content, products, and recommendations.

The UI should reinforce this distinction everywhere.


## 11. Flexible Delivery

A campaign should be able to deliver different kinds of content without requiring different campaign types.

Examples include:

- Messages
- Copyable text
- Resources
- Products
- Recommendations
- Creator Hub links

Creators should choose what they want to send rather than selecting from numerous campaign templates.
